Подробное описание

Класс для периодического выполнения задач по таймеру

TimerSchedule реализует механизм фонового выполнения callback-функций с заданным интервалом. Работает в отдельном потоке, обеспечивая неблокирующее выполнение периодических задач.

Заметки
При разрушении автоматически останавливает таймер и дожидается завершения рабочего потока. После вызова stop() или разрушения объекта повторный запуск невозможен. Для нового запуска требуется создание нового объекта.

Примерчик:

TimerSchedule timer;
timer.start(std::chrono::seconds(5), []() {
std::cout << "Прошло 5 секунд!" << std::endl;
});
std::this_thread::sleep_for(std::chrono::seconds(15));
timer.stop();

Запуск таймера

Запускает фоновый поток, который будет периодически вызывать callback-функцию с заданным интервалом. Если таймер уже запущен, повторный вызов игнорируется.

Аргументы
intervalИнтервал между вызовами callback-функции
callbackФункция, вызываемая по истечении каждого интервала
Заметки
Функция неблокирующая — запускает поток и возвращает управление.
Callback-функция выполняется в отдельном потоке, поэтому необходимо обеспечивать потокобезопасность доступа к общим данным.

Остановка таймера

Сигнализирует рабочему потоку о необходимости завершения и дожидается его окончания. После остановки таймер не может быть перезапущен — требуется создание нового объекта.

Заметки
Метод блокирующий — дожидается завершения рабочего потока.
Безопасен для многократного вызова.
